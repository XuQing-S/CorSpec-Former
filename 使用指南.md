# 使用指南

本文档用于说明如何在当前项目中完成 LIBS 金属牌号 6 分类实验，包括数据准备、配置文件训练、模型参数调整、SpeCorformer 消融实验、独立测试、输出结果查看和常见问题排查。

当前深度学习任务只针对金属牌号分类，模型输入主要为预处理后的单通道光谱张量 `[N, 1, 8192]`，输出为 6 类金属 logits `[N, 6]`。FNN 也支持使用特征工程后的表格特征 `[N, D]`。

## 1. 项目入口

常用入口如下：

1. 训练入口：`scr/dl/train.py`
2. 独立测试入口：`scr/dl/test.py`
3. 数据集封装：`scr/dl/dataset.py`
4. 模型实现目录：`scr/dl/models/`
5. 默认训练配置：`configs/default/`
6. 实验输出目录：`experiments/dl/`

当前已支持的模型包括：

1. `fnn`：全连接网络，适合融合特征或展平光谱基线。
2. `cnn`：轻量 1D CNN，适合作为主要卷积基线。
3. `lstm`：卷积降采样加双向 LSTM，适合序列依赖对比。
4. `resnet`：1D ResNet，适合更深的局部谱峰特征提取。
5. `vgg`：VGG 风格 1D CNN，适合堆叠卷积基线。
6. `inception`：多分支 1D Inception，适合多尺度卷积对比。
7. `convnext`：ConvNeXt 风格 1D 网络，适合现代卷积基线。
8. `transformer`：单尺度 patch-based Transformer。
9. `specorformer`：多尺度原始光谱 patch 嵌入加跨尺度交叉注意力模型。

## 3. 数据准备

训练默认读取预处理后的数据目录，例如：

1. `data/processed/balanced`
2. `data/processed/noise_shift/1`
3. `data/processed/noise_shift/2`
4. `data/processed/noise_shift/5`

光谱输入目录至少需要包含：

1. `train_x.npy`、`val_x.npy`、`test_x.npy`
2. `train_y_metal.npy`、`val_y_metal.npy`、`test_y_metal.npy`
3. `label_mapping.json`
4. 可选：`sample_index.csv`

光谱张量形状必须为：

```text
train_x.npy: [N_train, 1, 8192]
val_x.npy:   [N_val, 1, 8192]
test_x.npy:  [N_test, 1, 8192]
```

如果使用 FNN 的特征输入，还需要特征目录，例如 `data/features/balanced` 或 `data/features/noise_shift/5`。支持的特征文件包括：

1. `train_x_feature.npy`、`val_x_feature.npy`、`test_x_feature.npy`
2. `train_x_pca.npy`、`val_x_pca.npy`、`test_x_pca.npy`
3. `train_x_stat.npy`、`val_x_stat.npy`、`test_x_stat.npy`
4. `train_x_pls_metal.npy`、`val_x_pls_metal.npy`、`test_x_pls_metal.npy`

## 4. 快速训练

推荐优先使用配置文件启动训练。默认配置文件包括：

1. `configs/default/fnn.json`
2. `configs/default/cnn.json`
3. `configs/default/lstm.json`
4. `configs/default/resnet.json`
5. `configs/default/vgg.json`
6. `configs/default/inception.json`
7. `configs/default/convnext.json`
8. `configs/default/transformer.json`
9. `configs/default/specorformer.json`

直接运行默认配置：

```bash
python scr/dl/train.py --config configs/default/cnn.json
python scr/dl/train.py --config configs/default/resnet.json
python scr/dl/train.py --config configs/default/transformer.json
python scr/dl/train.py --config configs/default/specorformer.json
```

训练 FNN 融合特征基线：

```bash
python scr/dl/train.py --config configs/default/fnn.json
```

训练其他模型：

```bash
python scr/dl/train.py --config configs/default/lstm.json
python scr/dl/train.py --config configs/default/vgg.json
python scr/dl/train.py --config configs/default/inception.json
python scr/dl/train.py --config configs/default/convnext.json
```

注意：新模型建议通过配置文件中的 `model` 字段选择，不建议直接用命令行 `--model` 临时切换模型，以免命令行可选项与配置文件中的模型注册范围不一致。

## 5. 配置文件结构

训练配置通常包含以下部分：

```json
{
  "description": "experiment description",
  "data": {
    "processed_root": "data/processed/balanced",
    "features_root": "data/features/balanced",
    "input_kind": "spectrum",
    "feature_name": "feature"
  },
  "experiment": {
    "output_root": "experiments/dl",
    "run_name": "my_experiment"
  },
  "model": "cnn",
  "model_params": {},
  "training": {
    "epochs": 200,
    "batch_size": 128,
    "patience": 20,
    "class_weight": "none"
  },
  "optimization": {
    "learning_rate": 0.0003,
    "weight_decay": 0.0001,
    "lr_scheduler": "cosine",
    "lr_factor": 0.7,
    "lr_patience": 8,
    "min_lr": 0.00002
  },
  "runtime": {
    "device": "auto",
    "num_workers": 0,
    "random_seed": 42,
    "deterministic": false,
    "disable_progress": false
  },
  "visualization": {
    "no_data_figures": false,
    "visualize_max_samples": 3000,
    "saliency_max_samples": 80,
    "save_tsne": false
  }
}
```

配置说明：

1. `data.processed_root`：预处理光谱和标签目录。
2. `data.features_root`：特征工程输出目录，仅 `input_kind="feature"` 时使用。
3. `data.input_kind`：可选 `spectrum` 或 `feature`。
4. `data.feature_name`：可选 `feature`、`pca`、`stat`、`pls_metal`。
5. `experiment.output_root`：实验输出根目录。
6. `experiment.run_name`：当前实验目录名。
7. `model`：模型名称。
8. `model_params`：模型结构参数。
9. `training`：训练轮数、batch size、早停等。
10. `optimization`：学习率、权重衰减和调度器。
11. `runtime`：设备、随机种子、DataLoader worker 等。
12. `visualization`：可视化开关和抽样规模。

## 6. 命令行覆盖

配置文件中的字段会作为默认值，命令行参数优先级更高。

修改运行名称：

```bash
python scr/dl/train.py --config configs/default/cnn.json --run-name cnn_try_01
```

修改学习率：

```bash
python scr/dl/train.py --config configs/default/cnn.json --learning-rate 0.0005
```

修改训练轮数和 batch size：

```bash
python scr/dl/train.py --config configs/default/resnet.json --epochs 150 --batch-size 32
```

切换数据集：

```bash
python scr/dl/train.py \
  --config configs/default/cnn.json \
  --processed-root data/processed/noise_shift/1 \
  --run-name cnn_noise_shift_1
```

关闭数据分布类可视化：

```bash
python scr/dl/train.py --config configs/default/cnn.json --no-data-figures
```

保存 t-SNE 图：

```bash
python scr/dl/train.py \
  --config configs/default/cnn.json \
  --save-tsne \
  --visualize-max-samples 1000
```

临时覆盖模型结构参数：

```bash
python scr/dl/train.py \
  --config configs/default/specorformer.json \
  --model-params '{"self_attention_type": "mhsa", "pooling": "concat_mean"}' \
  --run-name specorformer_mhsa_concat
```

也可以把覆盖参数保存成单独 JSON 文件，再传给 `--model-params`。

## 7. 模型参数

模型结构参数都写在配置文件的 `model_params` 中。

### FNN

FNN 可以使用特征输入，也可以使用展平后的光谱输入。当前默认配置偏向特征输入。

```json
{
  "model": "fnn",
  "data": {
    "input_kind": "feature",
    "feature_name": "feature"
  },
  "model_params": {
    "input_length": 275,
    "hidden_dims": [128, 64, 32],
    "dropout": 0.25
  }
}
```

常调参数：

1. `input_length`：输入特征维度，feature 模式下训练脚本会根据训练特征自动覆盖。
2. `hidden_dims`：全连接层宽度。
3. `dropout`：分类头和隐藏层 dropout。

### CNN

```json
{
  "model": "cnn",
  "model_params": {
    "input_channels": 1,
    "channels": [32, 64, 128, 256],
    "kernel_sizes": [15, 9, 7, 5],
    "pool_sizes": [2, 2, 2, null],
    "dropout": 0.3
  }
}
```

常调参数：

1. `channels`：每层卷积通道数。
2. `kernel_sizes`：卷积核大小，较大卷积核更适合覆盖宽谱段。
3. `pool_sizes`：池化比例，`null` 表示该层不池化。
4. `dropout`：分类头 dropout。

### LSTM

```json
{
  "model": "lstm",
  "model_params": {
    "conv_channels": 64,
    "conv_stride": 4,
    "hidden_size": 128,
    "num_layers": 1,
    "bidirectional": true,
    "pooling": "mean",
    "dropout": 0.3
  }
}
```

常调参数：

1. `conv_stride`：进入 LSTM 前的降采样比例。
2. `hidden_size`：LSTM 隐状态维度。
3. `num_layers`：LSTM 层数。
4. `bidirectional`：是否使用双向 LSTM。
5. `pooling`：序列聚合方式，通常使用 `mean`。

### ResNet

```json
{
  "model": "resnet",
  "model_params": {
    "base_channels": 32,
    "blocks_per_stage": [2, 2, 2, 2],
    "stem_kernel_size": 7,
    "stem_stride": 2,
    "dropout": 0.3
  }
}
```

常调参数：

1. `base_channels`：初始通道数。
2. `blocks_per_stage`：每个 stage 的残差块数量。
3. `stem_kernel_size`：首层卷积核大小。
4. `stem_stride`：首层降采样比例。
5. `dropout`：分类头 dropout。

### Transformer

```json
{
  "model": "transformer",
  "model_params": {
    "input_channels": 1,
    "d_model": 128,
    "nhead": 4,
    "num_layers": 4,
    "dim_feedforward": 256,
    "dropout": 0.2,
    "patch_size": 16,
    "conv_kernel_size": 16,
    "pooling": "mean",
    "max_sequence_length": 520
  }
}
```

常调参数：

1. `d_model`：token 通道维度。
2. `nhead`：自注意力头数，必须整除 `d_model`。
3. `num_layers`：Transformer Encoder 层数。
4. `patch_size`：patch embedding 的步长。
5. `conv_kernel_size`：patch embedding 卷积核大小。
6. `pooling`：分类聚合方式，可选 `mean` 或 `cls`。
7. `max_sequence_length`：位置编码最大长度。

### SpeCorformer

SpeCorformer 是当前项目中的多尺度光谱 Transformer。它的核心流程是：

```text
原始光谱 X: [B, 1, 8192]
  ├─ 细尺度 PatchEmbed -> Self-Attention Encoder -> F1
  ├─ 中尺度 PatchEmbed -> Self-Attention -> Cross-Attention(F1) -> F2
  └─ 粗尺度 PatchEmbed -> Self-Attention -> Cross-Attention(F2) -> F3
分类头使用 F3 或多尺度均值拼接输出 logits
```

默认配置：

```json
{
  "model": "specorformer",
  "model_params": {
    "input_channels": 1,
    "embed_dims": [64, 128, 256],
    "patch_sizes": [4, 16, 64],
    "patch_kernel_sizes": [9, 33, 129],
    "depths": [2, 2, 4],
    "self_nheads": [2, 4, 8],
    "cross_nheads": [4, 8],
    "mlp_ratios": [4.0, 4.0, 4.0],
    "self_attention_type": "hfsa",
    "frequency_bands": [64, 64, 64],
    "local_kernel_sizes": [5, 5, 5],
    "cross_attention_type": "mhca",
    "dropout": 0.2,
    "pooling": "mean"
  }
}
```

关键参数含义：

1. `embed_dims`：每个尺度的 token 通道数。
2. `patch_sizes`：每个尺度从原始光谱切 patch 的步长。
3. `patch_kernel_sizes`：每个尺度的重叠卷积核大小，建议不小于对应 `patch_sizes`。
4. `depths`：每个尺度的块数；第一尺度是 Encoder 块，后续尺度是 Decoder-like 块。
5. `self_nheads`：尺度内 MHSA 头数；当 `self_attention_type` 为 `scsa`、`fdsa` 或 `hfsa` 时保留该参数用于快速切换 MHSA。
6. `cross_nheads`：跨尺度交叉注意力头数，长度必须等于尺度数减一。
7. `mlp_ratios`：每个尺度 MLP 隐层放大比例。
8. `self_attention_type`：可选 `mhsa`、`scsa`、`fdsa` 或 `hfsa`。
9. `frequency_bands`：FDSA 使用的频率上下文压缩带数，长度必须与尺度数一致。
10. `local_kernel_sizes`：HFSA 局部卷积分支的卷积核大小，长度必须与尺度数一致。
11. `cross_attention_type`：当前为 `mhca`，即标准多头交叉注意力。
12. `pooling`：可选 `mean`、`max` 或 `concat_mean`。

推荐 SpeCorformer 消融实验：

```bash
python scr/dl/train.py \
  --config configs/default/specorformer.json \
  --run-name specorformer_hfsa_mean

python scr/dl/train.py \
  --config configs/default/specorformer.json \
  --model-params '{"self_attention_type": "fdsa"}' \
  --run-name specorformer_fdsa_mean

python scr/dl/train.py \
  --config configs/default/specorformer.json \
  --model-params '{"self_attention_type": "scsa"}' \
  --run-name specorformer_scsa_mean

python scr/dl/train.py \
  --config configs/default/specorformer.json \
  --model-params '{"self_attention_type": "mhsa"}' \
  --run-name specorformer_mhsa_mean

python scr/dl/train.py \
  --config configs/default/specorformer.json \
  --model-params '{"pooling": "concat_mean"}' \
  --run-name specorformer_scsa_concat

python scr/dl/train.py \
  --config configs/default/specorformer.json \
  --model-params '{"patch_sizes": [8, 16, 64], "patch_kernel_sizes": [17, 33, 129]}' \
  --run-name specorformer_patch_8_16_64
```

调参建议：

1. 显存不足时优先增大第一尺度 `patch_sizes[0]`，例如从 `4` 调到 `8`。
2. 训练不稳定时降低 `learning_rate` 或增大 `dropout`。
3. 想保留更多细粒度窄峰信息时减小第一尺度 `patch_sizes[0]`。
4. 想增强跨尺度语义融合时增大 `cross_nheads` 或后两层 `depths`。
5. 对比注意力机制时只改 `self_attention_type`，保持其他参数不变。
6. 测试 FDSA 时可调整 `frequency_bands`，例如 `[32, 64, 64]` 或 `[64, 64, 128]`。
7. 测试 HFSA 时可调整 `local_kernel_sizes`，例如 `[3, 5, 7]` 用于逐尺度扩大局部峰感受野。

## 8. 训练输出

训练结果默认保存到：

```text
experiments/dl/<run_name>/
```

主要文件包括：

1. `config.json`：本次实验的完整配置和最终模型参数。
2. `metrics.json`：最佳验证集指标和测试集指标。
3. `history.csv`：每轮训练损失、验证损失、验证指标、学习率和耗时。
4. `classification_report.txt`：测试集分类报告。
5. `report.md`：实验摘要。
6. `checkpoints/best_model.pt`：验证集 Macro-F1 最佳模型。
7. `checkpoints/last_model.pt`：最后一轮模型。
8. `predictions/metal_<model>_test.csv`：测试集逐样本预测。
9. `confusion_matrix.csv`：测试集混淆矩阵。
10. `figures/`：训练曲线、混淆矩阵、PCA embedding、类别平均光谱、热图和 saliency 图。

重点查看顺序：

1. 先看 `metrics.json` 中的 `test.macro_f1` 和 `test.accuracy`。
2. 再看 `confusion_matrix_normalized.png`，确认主要混淆类别。
3. 然后看 `history.csv` 或 loss/F1 曲线，判断是否过拟合。
4. 最后看 PCA embedding、saliency 热图和类别平均光谱，辅助解释模型关注区域。

## 9. 独立测试

训练完成后，可以用 `scr/dl/test.py` 重新评估 checkpoint。

基本测试：

```bash
python scr/dl/test.py \
  --checkpoint experiments/dl/cnn_baseline/checkpoints/best_model.pt
```

脚本默认会从 checkpoint 上两级目录自动查找 `config.json`。如果需要手动指定：

```bash
python scr/dl/test.py \
  --checkpoint experiments/dl/cnn_baseline/checkpoints/best_model.pt \
  --config experiments/dl/cnn_baseline/config.json
```

指定测试输出目录：

```bash
python scr/dl/test.py \
  --checkpoint experiments/dl/cnn_baseline/checkpoints/best_model.pt \
  --output-dir experiments/dl/cnn_baseline/test_eval
```

保存测试集 PCA embedding：

```bash
python scr/dl/test.py \
  --checkpoint experiments/dl/cnn_baseline/checkpoints/best_model.pt \
  --save-embedding
```

切换测试数据集：

```bash
python scr/dl/test.py \
  --checkpoint experiments/dl/cnn_baseline/checkpoints/best_model.pt \
  --processed-root data/processed/noise_shift/1
```

测试输出默认保存到：

```text
experiments/dl/<run_name>/test_eval/
```

主要包括：

1. `test_metrics.json`
2. `classification_report.txt`
3. `test_report.md`
4. `predictions/`
5. `confusion_matrix.csv`
6. `figures/`

## 10. 推荐实验流程

建议按从简单到复杂的顺序建立基线：

1. 先训练 `cnn`，得到稳定卷积基线。
2. 训练 `fnn`，判断特征工程输入的上限。
3. 训练 `resnet`、`vgg`、`inception`、`convnext`，比较卷积结构差异。
4. 训练 `transformer`，得到单尺度注意力基线。
5. 训练 `specorformer`，比较多尺度跨尺度注意力是否提升。
6. 对 SpeCorformer 做 `scsa` 与 `mhsa` 消融。
7. 在 `balanced` 和 `noise_shift` 数据集上分别验证泛化能力。

推荐命令：

```bash
python scr/dl/train.py --config configs/default/cnn.json
python scr/dl/train.py --config configs/default/fnn.json
python scr/dl/train.py --config configs/default/resnet.json
python scr/dl/train.py --config configs/default/transformer.json
python scr/dl/train.py --config configs/default/specorformer.json
```

噪声偏移数据集示例：

```bash
python scr/dl/train.py \
  --config configs/default/specorformer.json \
  --processed-root data/processed/noise_shift/1 \
  --run-name specorformer_noise_shift_1
```

## 11. 常见问题

### ModuleNotFoundError: No module named 'torch'

说明当前环境没有安装 PyTorch，或当前终端没有进入正确的 Python 环境。请先安装 PyTorch，并确认：

```bash
python - <<'PY'
import torch
print(torch.__version__)
PY
```

### 配置文件参数没有生效

检查以下几点：

1. 是否使用了正确的 `--config` 路径。
2. 参数是否写在正确字段中，例如模型结构应写在 `model_params`。
3. 是否又通过命令行传入了同名参数，命令行优先级更高。
4. JSON 是否格式正确，可用 `python -m json.tool <config>` 检查。

### 训练很慢

可尝试：

1. 使用 GPU：`--device cuda`。
2. 降低 `batch_size`。
3. 减小模型宽度，例如 `channels`、`embed_dims` 或 `hidden_size`。
4. 关闭 t-SNE：不要使用 `--save-tsne`。
5. 关闭数据分布类图：使用 `--no-data-figures`。
6. 对 SpeCorformer 增大第一尺度 `patch_sizes[0]`。

### 显存不足

可尝试：

1. 降低 `--batch-size`。
2. 减小 CNN 的 `channels`。
3. 减小 ResNet 的 `base_channels` 或 `blocks_per_stage`。
4. 减小 LSTM 的 `hidden_size` 或 `num_layers`。
5. 减小 Transformer 的 `d_model`、`num_layers` 或增大 `patch_size`。
6. 减小 SpeCorformer 的 `embed_dims`、`depths`，或增大 `patch_sizes`。

### 输出目录被覆盖

如果多次使用相同 `run_name`，输出会写到同一目录。建议每次实验使用不同运行名：

```bash
python scr/dl/train.py \
  --config configs/default/cnn.json \
  --run-name cnn_baseline_02
```

### FNN 特征输入维度不匹配

当 `input_kind="feature"` 时，训练脚本会根据训练特征矩阵自动设置 FNN 的 `input_length`。如果仍然报错，请检查：

1. `features_root` 是否指向正确目录。
2. `feature_name` 是否为 `feature`、`pca`、`stat` 或 `pls_metal`。
3. 特征文件和标签文件的样本数量是否一致。

### SpeCorformer 参数长度报错

SpeCorformer 的多尺度参数长度必须匹配：

1. `embed_dims`、`patch_sizes`、`patch_kernel_sizes`、`depths`、`self_nheads`、`mlp_ratios` 长度相同。
2. `cross_nheads` 长度等于尺度数量减一。
3. `patch_kernel_sizes[i]` 建议大于或等于 `patch_sizes[i]`。
4. 使用 `self_attention_type="mhsa"` 时，`embed_dims[i]` 必须能被 `self_nheads[i]` 整除。
5. 交叉注意力中，当前尺度 `embed_dims[i]` 必须能被 `cross_nheads[i - 1]` 整除。

## 12. 结果复现实验建议

为了让实验可复现，建议：

1. 每次实验使用唯一 `run_name`。
2. 保留训练输出目录中的 `config.json`。
3. 对重要实验记录 Git commit 或至少记录当前代码状态。
4. 固定 `random_seed`。
5. 对同一模型至少重复 3 次不同 seed，比较平均值和方差。
6. 报告结果时同时给出 Accuracy、Macro-F1、混淆矩阵和主要错误类别。
